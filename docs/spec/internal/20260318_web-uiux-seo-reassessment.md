| 4 | INP 最適化 | 未計測 | フィルタ・カルーセル操作の応答時間を 200ms 以下に最適化 | 中 |

### 2. 機能面での UX 改善提案

以下は「あったらいいな」の機能提案で、ユーザー体験とコンバージョンの向上を目的とする。

#### 2-1. サイト内検索（web/user）

**現状:** ナビゲーションの検索がコメントアウトされており、未実装。

**提案:**
```text
検索バーをヘッダーに常設
  ├── 商品名・説明文のキーワード検索
  ├── カテゴリ・タグによるファセット検索
  ├── 検索候補のサジェスト表示（インクリメンタルサーチ）
  └── 検索結果ページ（ページネーション + ソート）

期待効果: 検索利用者のコンバージョン率は通常の 2〜3 倍
```

#### 2-2. 商品フィルタリング・ソート強化（web/user）

**現状:** ソート機能（新着順・価格順）は実装済み。フィルタリングは未実装。

**提案:**
```text
フィルタリングオプション:
  ├── カテゴリ（野菜・果物・海産物・加工品 等）
  ├── 産地（都道府県別）
  ├── 価格帯（〜1,000円 / 1,000〜3,000円 / 3,000〜5,000円 / 5,000円〜）
  ├── 配送タイプ（常温・冷蔵・冷凍）
  └── 在庫あり/なし

モバイル: ボトムシート型フィルタパネル（サムゾーンに配置）
デスクトップ: サイドバー型フィルタパネル

期待効果: フィルタ利用者の購入率 1.5 倍
```

#### 2-3. ウィッシュリスト / お気に入り（web/user）

**現状:** 未実装。

**提案:**
```text
仕様:
  ├── 商品カード・詳細ページにハートアイコンを追加
  ├── ログイン時: API に保存（デバイス間同期）
  ├── 未ログイン時: localStorage に保存
  ├── マイページにお気に入り一覧を表示
  └── 在庫復活・値下げ時にメール/LINE 通知

期待効果: 再訪率の向上、カート放棄からの復帰
```

#### 2-4. 最近チェックした商品（web/user）

**現状:** 未実装。

**提案:**
```text
仕様:
  ├── 商品詳細ページの閲覧履歴を localStorage に保存（最新 20 件）
  ├── トップページ・商品一覧ページ下部にカルーセル表示
  └── 「もう一度見る」セクションとして表示

期待効果: 回遊率向上、衝動買いの促進
```

#### 2-5. おすすめ商品のパーソナライズ（web/user）

**現状:** おすすめセクションが `v-if="false"` で非表示。

**提案:**
```text
段階的実装:
  Phase A: ルールベース（同じコーディネーターの商品、同カテゴリの人気商品）
  Phase B: 行動ベース（閲覧・購入履歴に基づく協調フィルタリング）
  Phase C: AI ベース（LLM を活用した商品説明生成 + レコメンデーション）

配置箇所:
  ├── 商品詳細ページ下部（「こちらもおすすめ」）
  ├── カート内（「一緒に買われている商品」）
  └── トップページ（「あなたにおすすめ」）

期待効果: AI パーソナライズ導入企業は CVR 25〜30% 向上（業界平均）
```

#### 2-6. チェックアウト UX の改善（web/user + web/liff）

**現状:** カート → 決済 → 完了の基本フローは実装済み。

**提案:**
```text
改善案:
  ├── 進捗インジケーター（Step 1: カート確認 → Step 2: 配送先 → Step 3: 決済）
  ├── ワンクリック購入（リピーター向け、配送先・決済方法を保存）
  ├── クレジットカードのリアルタイムバリデーション（カード種別自動判定）
  ├── 決済方法選択の拡充（PayPay, LINE Pay, Apple Pay）
  ├── 高額商品への分割払い表示（¥10,000 以上で「月々 ¥X から」を表示）
  └── 注文確認モーダル（決済前の最終確認画面）

期待効果:
  - ワンクリック: チェックアウト時間 30% 短縮 → CVR 5〜10% 向上
  - 分割払い: 高単価商品の購入率 12〜15% 向上
```

#### 2-7. レビュー・UGC（ユーザー生成コンテンツ）の強化（web/user）

**現状:** レビュー投稿・表示機能は実装済み。一覧ページでの星評価表示は未実装。

**提案:**
```text
改善案:
  ├── 商品一覧カードに星評価を表示（一目で評判がわかる）
  ├── 写真付きレビューの投稿・表示（開封写真、調理写真）
  ├── レビュー投稿でポイント付与（「レビューを書いて 100pt」）
  ├── 「参考になった」ボタン（レビューの信頼性可視化）
  ├── 購入済みバッジ（「購入者レビュー」の表示）
  └── 商品ページに「最近 X 人が購入」のリアルタイム表示

期待効果:
  - 68% の消費者は 4 つ星以上の商品しか検討しない
  - UGC 写真は専門撮影より 81% 効果的（2026 年調査）
```

#### 2-8. 数量変更 UX の改善（web/liff）

**現状:** カート内で数量変更不可（削除して再追加が必要）。

**提案:**
```text
改善案:
  ├── カート内に +/- ボタン付き数量ピッカーを追加
  ├── 在庫上限を超えた場合のリアルタイムフィードバック
  └── スワイプで削除（モバイル操作の効率化）

期待効果: カート操作のストレス軽減、購入単価の向上
```

#### 2-9. PWA 対応（web/user）

**現状:** 未対応（manifest.json なし、Service Worker なし）。

**提案:**
```text
仕様:
  ├── Web App Manifest の追加（ホーム画面追加対応）
  ├── Service Worker によるオフラインフォールバック
  ├── プッシュ通知（新商品・ライブ配信開始の通知）
  └── アプリライクな起動体験（スプラッシュスクリーン）

期待効果: リテンション率の向上、エンゲージメント向上
```

#### 2-10. 産地・生産者ストーリーの強化（web/user）

**現状:** コーディネーター・生産者のプロフィール表示は実装済み。

**提案:**
```text
改善案:
  ├── 生産者の「ものづくりストーリー」ページ（写真 + テキスト + 動画）
  ├── 産地マップ（Google Maps 連携、地域別の商品・体験を表示）
  ├── 旬のカレンダー（月別のおすすめ産地・食材を表示）
  ├── 生産者をフォローする機能（新商品追加時に通知）
  └── 生産者ランキング（レビュー数・評価順）

期待効果:
  - 地域特産品 EC の差別化要因
  - 日本の消費者は「作り手の顔が見える」商品に高い信頼を置く
  - ストーリー閲覧者の購入率は非閲覧者の 2 倍（業界統計）
```
