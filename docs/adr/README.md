# Architecture Decision Records (ADR)

このディレクトリには、Furumaru プロジェクトのアーキテクチャに関する意思決定記録を格納します。

## ADR とは

Architecture Decision Record (ADR) は、アーキテクチャ上の重要な意思決定とその背景・根拠を記録するドキュメントです。
将来の開発者が「なぜこの設計判断がなされたか」を理解するための参照資料として機能します。

## ADR の作成方法

1. [テンプレート](./template.md) をコピーして新規ファイルを作成する
2. ファイル名は `NNN-<kebab-case-title>.md` (例: `007-new-decision.md`)
3. 連番は既存 ADR の最大番号 + 1 とする
4. 上記の ADR 一覧テーブルに新規 ADR を追加する

## ステータスの定義

| ステータス | 説明 |
|-----------|------|
| 提案 | レビュー中の提案 |
| 承認済み | チームで合意・承認された決定 |
| 廃止 | 後続の ADR により置き換えられた決定 |
| 保留 | 検討を一時中断している決定 |
